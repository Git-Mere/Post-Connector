# GitHub README 지시사항

## 섹션 구조 (이 순서로 작성)

1. `# 프로젝트명` + 한 줄 태그라인
2. 배지 — 라이선스, 언어 (선택)
3. `## Features` — 주요 기능 bullet list 3–5개
4. `## Demo` — 스크린샷 자리 표시자 `![screenshot](./screenshot.png)` 또는 라이브 링크 (있으면)
5. `## Built With` — 기술 스택. 각 항목에 선택 이유 한 줄 추가. `userInput.problemSolved`가 있으면 맥락으로 활용.
   형식: `- **기술명** — 선택 이유 한 줄`
6. `## Getting Started` — Prerequisites + Installation + 실행 커맨드 (코드 블록)
7. `## What I Learned` — `userInput.learnings`가 있으면 그 내용을 "문제 → 접근법 → 결과" 구조로 서술. 없으면 코드에서 추론 가능한 기술적 도전을 서술.
   - "React를 배웠다" 같은 추상적 한 줄 서술 금지
   - 구체적 상황 묘사 필수
8. `## License`

## 언어

- 전역 지시의 "README 언어 기준" 규칙을 무시하고 **항상 영어로 작성한다** (포트폴리오가 영어 기준). 기존 README가 한국어여도 영어로 출력.

## 원칙

- `userInput` 필드가 채워져 있으면 반드시 활용한다.
- 채워지지 않은 필드는 project data에서 추론한다. 확인되지 않은 내용을 지어내지 않는다.
- 기존 README 덮어쓰지 않는다. PR 또는 새 브랜치로 제안.
