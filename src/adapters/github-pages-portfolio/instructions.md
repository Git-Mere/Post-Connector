# GitHub Pages Portfolio instructions

- Keep tagline to one sentence; lead with what the project does, not the tech stack.
- description must follow problem → solution → impact order. Avoid generic openers like "This project...".
- tags should reflect the actual languages and frameworks used — do not list every dependency, only the ones that define the project.
- imageAlt should describe what a viewer would see (e.g., "dashboard showing real-time metrics" or "architecture diagram of pipeline").
