# GitHub Pages Portfolio instructions

- title should match the repo name or a clear human-readable version of it.
- tech should reflect the actual languages and frameworks — do not list every dependency, only the ones that define the project.
- For "other" type: keep description to one or two sentences; lead with what the project does, not the tech stack.
- For "featured" type: description must follow problem → solution → impact order. Avoid generic openers like "This project...".
