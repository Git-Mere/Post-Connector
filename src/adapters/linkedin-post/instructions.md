# LinkedIn Post 지시사항

## 구조 (이 순서로 작성)

1. 훅 (1-2줄) — "더 보기" 이전 노출 영역. 프로젝트 핵심 + 독자가 왜 읽어야 하는지. 구체적일수록 좋음.
2. 맥락 — 왜 만들었나, 어떤 문제를 해결하나. userInput.problemSolved가 있으면 활용.
3. 핵심 도전 + 배운 것 — 구체적 기술 챌린지와 해결 방법. userInput.learnings가 있으면 활용. "React를 배웠다" 수준 금지 — 문제 → 접근법 → 결과 구조.
4. 기술 스택 — 핵심 기술만 1-2줄.
5. GitHub 링크 (+ 데모 링크 있으면) — 마지막 줄에.
6. 해시태그 — 관련 기술 + 일반 태그 3-5개, 맨 끝.

## 길이
- 목표: 900-1300자
- 절대 최대: 3000자

## 원칙
- plain text만. ** ## `` 등 마크다운 문법 일절 사용 금지.
- 빈 줄로 문단 구분.
- 이모지는 사용자가 요청하지 않으면 사용 안 함.
- 확인되지 않은 수치/성과 지어내지 말 것.
- 링크는 마지막 줄에만.
